# اختبارات Etqan API

## تشغيل الاختبارات

**تشغيل الأوامر من مجلد الباكند:** `E:\etqanfull\Etqan` (وليس من داخل مجلد فرعي مثل `Etqan\Etqan`).

1. **تأكد من تشغيل قاعدة البيانات (MySQL)** وتطبيق الـ migrations:
   ```bash
   cd E:\etqanfull\Etqan
   npx prisma migrate deploy
   ```

2. **تشغيل الـ seed** (لإنشاء أدمن، أطباء، مرضى، وجبات، تمارين، قناة):
   ```bash
   npm run prisma:seed
   ```

   إذا ظهر تحذير في الاختبار عن خطة التمارين (مثل `repMin` أو `exerciseId`)، تأكد من تطبيق كل الـ migrations:
   ```bash
   npx prisma migrate deploy
   ```

3. **تشغيل الاختبارات**:
   ```bash
   npm test
   ```

   أو في وضع المراقبة:
   ```bash
   npm run test:watch
   ```

## ما الذي تغطيه الاختبارات؟

ملف `integration/user-doctor-flow.test.js` يختبر المسار الكامل:

| # | الخطوة |
|---|--------|
| 1 | إنشاء مستخدم (مريض) من الأدمن ثم تسجيل دخوله |
| 2 | الأدمن يحدد للمستخدم دكتوراً |
| 3 | المستخدم يدخل الشات مع الدكتور ويرسل رسالة |
| 4 | الدكتور يرد في الشات |
| 5 | الدكتور يضيف ملاحظة للمريض |
| 6 | الدكتور يضيف خطة أسبوعية للوجبات |
| 7 | الدكتور يضيف خطة أسبوعية للتمارين |
| 8 | المستخدم يعرض خطط التغذية والتمارين |
| 9 | مستخدمان يرسلان رسائل في قناة واحدة (شات القنوات) |
| 10 | عرض رسائل القناة |
